# 个人博客项目重新规划方案

## 1. 项目概述

### 1.1 当前项目状态

基于现有代码库分析，当前项目是一个基于 **Vue 3 + Vite** 的个人博客应用，包含以下特性：

| 维度 | 现状 |
|------|------|
| **技术栈** | Vue 3 + Vite + Vue Router + Ant Design Vue + GSAP |
| **页面数量** | 7个页面（首页、文章列表、文章详情、关于、归档、工具、监控） |
| **数据来源** | 本地 Mock 数据 |
| **设计风格** | 玻璃拟态 + 深色主题 |

### 1.2 用户反馈与痛点

根据用户反馈，当前项目存在以下问题：
- ✖️ 首页展示性强但实用性不足
- ✖️ 部分功能界面不够美观
- ✖️ 工具页功能单薄（仅工资计算器）
- ✖️ 监控页功能待完善

---

## 2. 项目定位与目标

### 2.1 核心定位

> **技术博客 + 工具平台**
> 
> 一个面向开发者的个人技术博客，提供高质量技术文章阅读体验，同时集成实用的开发工具。

### 2.2 目标用户

| 用户角色 | 需求描述 |
|----------|----------|
| **技术读者** | 阅读高质量技术文章、学习前端技术、获取开发技巧 |
| **开发者** | 使用实用工具（代码格式化、颜色转换等）、查看系统监控 |
| **博主** | 展示个人技术成果、分享技术经验 |

### 2.3 核心价值

- 📚 **优质内容**：精选前端技术文章
- 🔧 **实用工具**：开发效率工具集
- 🎨 **现代设计**：美观、流畅的用户体验
- 📱 **全平台适配**：响应式设计，支持多端

### 3.2 各模块详细规划

#### 模块一：首页 (Home)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **Hero 区域** | 品牌展示 + CTA 按钮 | 高 |
| **文章预览** | 最新文章卡片（带封面图） | 高 |
| **数据统计** | 文章数、阅读量、粉丝数等 | 中 |
| **热门标签** | 标签云展示 | 中 |
| **动态内容** | 近期更新、阅读排行 | 中 |

#### 模块二：文章中心 (Articles)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **文章列表** | 卡片式布局，支持筛选/排序 | 高 |
| **分类导航** | 按技术分类（Vue、React、性能等） | 高 |
| **搜索功能** | 全文搜索文章标题/内容 | 高 |
| **阅读统计** | 阅读量、点赞数、评论数 | 中 |
| **文章详情** | Markdown 渲染、代码高亮、目录导航 | 高 |

#### 模块三：工具库 (Tools)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **代码格式化器** | 支持多种语言代码格式化 | 高 |
| **颜色工具** | 颜色转换、渐变色生成器 | 高 |
| **Markdown 预览** | 实时 Markdown 编辑器 | 高 |
| **时间戳转换** | Unix 时间戳转换 | 中 |
| **工资计算器** | 保留现有功能并优化 | 中 |

#### 模块四：归档 (Archive)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **时间轴视图** | 按年份/月份归档 | 高 |
| **标签筛选** | 按标签筛选文章 | 中 |
| **统计图表** | 文章发布趋势 | 低 |

#### 模块五：关于我 (About)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **个人介绍** | 个人简介、技术栈展示 | 高 |
| **项目经验** | 工作经历、项目展示 | 高 |
| **联系方式** | 社交链接、邮箱 | 中 |

#### 模块六：系统监控 (Monitor)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **资源监控** | CPU、内存、磁盘使用情况 | 高 |
| **网络状态** | 网络接口信息 | 中 |
| **进程列表** | 系统进程监控 | 低 |

## 4. 使用场景分析

### 4.1 场景一：技术文章阅读

**用户旅程**：
1. 用户访问首页 → 浏览文章预览
2. 点击感兴趣的文章 → 进入文章详情页
3. 阅读文章内容 → 查看代码示例
4. 通过目录导航快速跳转 → 阅读完成

**核心需求**：良好的阅读体验、代码高亮、目录导航

### 4.2 场景二：使用开发工具

**用户旅程**：
1. 用户进入工具页 → 选择需要的工具
2. 输入数据 → 获得结果
3. 复制结果 → 返回开发工作

**核心需求**：简洁易用、响应快速、支持复制

### 4.3 场景三：浏览归档内容

**用户旅程**：
1. 用户进入归档页 → 选择年份/月份
2. 浏览历史文章 → 点击阅读

**核心需求**：清晰的时间线、快速定位

### 4.4 场景四：了解博主

**用户旅程**：
1. 用户访问关于页 → 了解博主背景
2. 查看技术栈 → 查看项目经验
3. 获取联系方式 → 建立联系

**核心需求**：信息清晰、专业形象

## 5. 技术架构规划

### 5.1 技术栈选择

| 分类 | 技术 | 版本 | 说明 |
|------|------|------|------|
| **框架** | Vue 3 | ^3.5 | 渐进式 JavaScript 框架 |
| **构建工具** | Vite | ^8.0 | 下一代前端构建工具 |
| **路由** | Vue Router | ^5.0 | Vue 官方路由管理器 |
| **UI组件** | Ant Design Vue | ^4.2 | 企业级 UI 组件库 |
| **动画** | GSAP | ^3.15 | 高性能动画库 |
| **图表** | ECharts | ^6.1 | 数据可视化图表库 |
| **代码高亮** | Prism.js | ^1.29 | 轻量级代码高亮 |

### 5.3 状态管理策略

由于项目复杂度适中，采用 **响应式 ref/reactive** 配合 **localStorage** 进行状态管理：

| 状态类型 | 存储方式 | 说明 |
|----------|----------|------|
| **全局状态** | localStorage | 用户偏好、主题设置 |
| **页面状态** | ref/reactive | 搜索关键词、筛选条件 |
| **路由状态** | Vue Router | 当前路由、参数 |

## 6. 视觉设计规范

### 6.1 设计风格

- **主色调**：渐变色（蓝紫系）
- **背景**：深色主题为主，支持浅色切换
- **卡片**：圆角设计、柔和阴影
- **字体**：无衬线字体，清晰可读

### 6.2 色彩方案

| 颜色类型 | 值 | 用途 |
|----------|-----|------|
| **主色** | `#6366f1` (Indigo) | 按钮、链接、高亮 |
| **次色** | `#8b5cf6` (Violet) | 渐变、装饰 |
| **背景** | `#0f0f1a` | 页面背景 |
| **卡片** | `#1a1a2e` | 卡片背景 |
| **文字** | `#e2e8f0` | 正文 |
| **边框** | `#2d2d44` | 分隔线 |

## 8. 风险评估与应对

| 风险 | 可能性 | 影响 | 应对策略 |
|------|--------|------|----------|
| 功能过于复杂 | 中 | 开发周期延长 | 分阶段迭代，优先核心功能 |
| 性能问题 | 低 | 用户体验下降 | 使用虚拟滚动、懒加载 |
| 兼容性问题 | 低 | 部分用户无法访问 | 确保响应式设计，测试多浏览器 |
| 数据管理 | 中 | Mock 数据维护困难 | 建立规范的数据结构 |

---

## 9. 总结

本规划旨在将当前博客项目从"展示型"升级为"实用型"，重点提升：

1. **内容组织**：优化文章展示和筛选
2. **功能实用性**：扩展工具库功能
3. **用户体验**：现代设计风格、流畅交互
4. **技术架构**：模块化、可扩展的代码结构

建议按照 **Phase 1 → Phase 5** 的顺序逐步实施，确保每个阶段都有可交付的成果。
