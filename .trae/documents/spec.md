# 个人博客项目规格说明文档

## 1. 需求分析

### 1.1 用户反馈摘要

根据用户反馈，当前项目存在以下核心问题：

| 问题编号 | 问题描述 | 严重程度 |
|----------|----------|----------|
| REQ-001 | 首页展示性强但实用性不足 | 高 |
| REQ-002 | 部分功能界面不够美观 | 中 |
| REQ-003 | 工具页功能单薄（仅工资计算器） | 高 |
| REQ-004 | 监控页功能待完善 | 中 |

### 1.2 用户角色与需求

| 用户角色 | 核心需求 | 业务目标 |
|----------|----------|----------|
| **技术读者** | 阅读高质量技术文章 | 获取知识、学习技术 |
| **开发者** | 使用实用开发工具 | 提高开发效率 |
| **博主** | 展示个人技术成果 | 建立专业形象 |
| **访客** | 浏览内容、了解博主 | 获取信息、建立联系 |

### 1.3 功能需求矩阵

| 需求 | 描述 | 来源 |
|------|------|------|
| FR-001 | 首页展示最新文章列表 | REQ-001 |
| FR-002 | 文章搜索功能 | REQ-001 |
| FR-003 | 文章分类导航 | REQ-001 |
| FR-004 | 代码格式化工具 | REQ-003 |
| FR-005 | 颜色转换工具 | REQ-003 |
| FR-006 | Markdown预览工具 | REQ-003 |
| FR-007 | 系统资源监控 | REQ-004 |
| FR-008 | 美观的UI设计 | REQ-002 |

## 2. 功能规格

### 2.1 首页 (Home)

**功能概述**：作为博客入口，展示核心内容和数据统计

**子功能**：

| 功能 | 描述 | 优先级 |
|------|------|--------|
| Home-001 | Hero区域：品牌标语 + CTA按钮 | 高 |
| Home-002 | 最新文章卡片展示（带封面图） | 高 |
| Home-003 | 数据统计面板（文章数、阅读量、标签数） | 中 |
| Home-004 | 热门标签云 | 中 |
| Home-005 | 阅读排行榜 | 中 |

**设计约束**：
- Hero区域需突出品牌调性
- 卡片布局响应式适配
- 统计数据实时更新

### 2.2 文章中心 (Articles)

**功能概述**：文章列表与详情展示

**子功能**：

| 功能 | 描述 | 优先级 |
|------|------|--------|
| Art-001 | 文章卡片列表（双栏布局） | 高 |
| Art-002 | 分类筛选导航 | 高 |
| Art-003 | 关键词搜索 | 高 |
| Art-004 | 排序功能（时间/阅读量） | 高 |
| Art-005 | 文章详情页（Markdown渲染） | 高 |
| Art-006 | 目录导航 | 中 |
| Art-007 | 代码高亮显示 | 高 |

**设计约束**：
- 搜索支持标题、摘要、标签匹配
- 文章详情支持移动端阅读

### 2.3 工具库 (Tools)

**功能概述**：开发效率工具集合

**子功能**：

| 功能 | 描述 | 优先级 |
|------|------|--------|
| Tool-001 | 代码格式化器（支持JS/TS/Vue/React） | 高 |
| Tool-002 | 颜色转换工具（HEX/RGB/HSL） | 高 |
| Tool-003 | 渐变色生成器 | 中 |
| Tool-004 | Markdown实时预览 | 高 |
| Tool-005 | 时间戳转换器 | 中 |
| Tool-006 | 工资计算器（保留现有） | 中 |

**设计约束**：
- 工具卡片式展示
- 支持一键复制结果
- 响应式布局

### 2.4 归档页 (Archive)

**功能概述**：文章时间线归档

**子功能**：

| 功能 | 描述 | 优先级 |
|------|------|--------|
| Arch-001 | 年份/月份时间轴 | 高 |
| Arch-002 | 标签筛选 | 中 |
| Arch-003 | 文章发布趋势图表 | 低 |

### 2.5 关于页 (About)

**功能概述**：个人介绍与联系方式

**子功能**：

| 功能 | 描述 | 优先级 |
|------|------|--------|
| About-001 | 个人简介 | 高 |
| About-002 | 技术栈展示 | 高 |
| About-003 | 项目经验 | 高 |
| About-004 | 社交链接 | 中 |

### 2.6 监控页 (Monitor)

**功能概述**：系统资源监控面板

**子功能**：

| 功能 | 描述 | 优先级 |
|------|------|--------|
| Mon-001 | CPU使用率监控 | 高 |
| Mon-002 | 内存使用监控 | 高 |
| Mon-003 | 磁盘空间监控 | 中 |
| Mon-004 | 网络状态监控 | 中 |

## 3. UI/UX 规格

### 3.1 设计风格

| 维度 | 规格 |
|------|------|
| 主题 | 深色主题为主，支持浅色切换 |
| 配色 | 蓝紫渐变系（#6366f1 → #8b5cf6） |
| 字体 | Inter/思源黑体，无衬线 |
| 卡片 | 圆角16px，柔和阴影 |
| 间距 | 8px基准间距系统 |

### 3.2 交互规范

| 交互类型 | 规范 |
|----------|------|
| 悬停效果 | 卡片抬升、阴影增强 |
| 过渡动画 | 300ms ease-out |
| 加载状态 | Skeleton骨架屏 |
| 错误状态 | 友好提示 + 重试按钮 |
| 空状态 | 图标 + 说明文字 |

### 3.3 响应式断点

| 断点 | 宽度 | 布局 |
|------|------|------|
| 移动端 | < 768px | 单列布局 |
| 平板端 | 768px - 1024px | 自适应布局 |
| 桌面端 | > 1024px | 双栏布局 |

## 4. 技术规格

### 4.1 技术栈

| 分类 | 技术 | 版本 |
|------|------|------|
| 框架 | Vue 3 | ^3.5 |
| 构建工具 | Vite | ^8.0 |
| 路由 | Vue Router | ^5.0 |
| UI组件 | Ant Design Vue | ^4.2 |
| 动画 | GSAP | ^3.15 |
| 图表 | ECharts | ^6.1 |
| 图标 | Ant Design Icons | ^7.0 |

### 4.2 架构设计

```
┌─────────────────────────────────────────────┐
│              Presentation Layer             │
│  Views (Home, Articles, Tools, ...)        │
│  Components (Layout, ArticleCard, ...)     │
├─────────────────────────────────────────────┤
│                Business Layer               │
│  Composables (useArticle, useTheme, ...)   │
│  Utils (formatters, validators)            │
├─────────────────────────────────────────────┤
│                 Data Layer                  │
│  Mock Data (articles.js)                   │
└─────────────────────────────────────────────┘
```

### 4.3 文件结构

```
src/
├── views/                    # 页面视图
│   ├── Home.vue             # 首页
│   ├── Articles.vue         # 文章列表
│   ├── ArticleDetail.vue    # 文章详情
│   ├── Tools.vue            # 工具页
│   ├── Archive.vue          # 归档页
│   ├── About.vue            # 关于页
│   └── Monitor.vue          # 监控页
├── components/               # 公共组件
│   ├── Layout.vue           # 布局组件
│   ├── ArticleCard.vue      # 文章卡片
│   ├── StatCard.vue         # 统计卡片
│   ├── TagCloud.vue         # 标签云
│   └── ToolCard.vue         # 工具卡片
├── composables/              # 组合式函数
│   ├── useArticleMeta.js    # 文章元数据
│   ├── useTheme.js          # 主题切换
│   └── useTools.js          # 工具函数
├── mock/                     # 模拟数据
│   └── articles.js          # 文章数据
├── router/                   # 路由配置
│   └── index.js             # 路由定义
├── utils/                    # 工具函数
│   └── formatters.js        # 格式化工具
├── App.vue                  # 根组件
├── main.js                  # 入口文件
└── style.css                # 全局样式
```

## 5. 数据规格

### 5.1 文章数据结构

```json
{
  "id": number,
  "title": string,
  "summary": string,
  "content": string,
  "tags": string[],
  "date": string (YYYY-MM-DD),
  "readingTime": number (minutes),
  "views": number,
  "coverColor": string (gradient)
}
```

### 5.2 工具数据结构

```json
{
  "id": string,
  "name": string,
  "description": string,
  "icon": string,
  "category": string,
  "featured": boolean
}
```

### 5.3 Mock数据规范

- 文件位置：`src/mock/articles.js`
- 编码格式：UTF-8
- 数据量：至少10篇示例文章

## 6. 安全性考虑

### 6.1 XSS防护

| 措施 | 说明 |
|------|------|
| 内容过滤 | 对用户输入进行HTML转义 |
| 代码高亮 | 使用安全的代码高亮库 |
| 渲染限制 | 禁止执行未知脚本 |

### 6.2 资源保护

| 措施 | 说明 |
|------|------|
| 图片懒加载 | 防止资源滥用 |
| 请求限流 | 避免恶意请求 |
| 缓存策略 | 合理设置缓存时间 |
